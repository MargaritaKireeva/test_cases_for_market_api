# Тест-кейсы: Аутентификация и Регистрация

## Регистрация (POST /auth/register)

### 1. Успешная регистрация пользователя с валидными email и паролем

**Предусловие:** Сервер запущен. БД пуста (пользователь с указанным email отсутствует).

**Шаги:**
1. Отправить POST-запрос на `/auth/register` с телом из набора данных

**Ожидаемый результат:**
- Код ответа: `201 Created`
- В теле ответа присутствуют поля `id` (целое число) и `email` (строка)
- Поле `email` в ответе совпадает с переданным
- В БД создана запись пользователя: `id` и `email` совпадают с ответом, `password` содержит хеш пароля

**Наборы данных:**
- `email = "user@mail.com"`, `password = "123456"`
- `email = "u12349@gmail.com"`, `password = "1234567"`
- `email = "95969ODJ@yandex.ru"`, `password = "111111111111111111111111111111"` (30 символов)

---

### 2. Регистрация с уже существующим email

**Предусловие:** Сервер запущен. БД пуста.

**Шаги:**
1. Отправить POST-запрос на `/auth/register` с телом `{"email": "user@mail.ru", "password": "password"}`
2. Отправить POST-запрос на `/auth/register` с телом `{"email": "user@mail.ru", "password": "password"}`

**Ожидаемый результат:**
- Первый запрос: код `201 Created`, пользователь создан
- Второй запрос: код `409 Conflict`, тело `{"detail": "User already exists"}`

---

### 3. Регистрация с некорректным email

**Предусловие:** Сервер запущен.

**Шаги:**
1. Отправить POST-запрос на `/auth/register` с телом `{"email": "<email>", "password": "password"}`

**Ожидаемый результат:**
- Код ответа: `422 Unprocessable Entity`
- В `detail[0].msg` содержится текст `"value is not a valid email address"`

**Наборы данных (email):**
- `""`
- `"1"`
- `"mail.ru"`
- `"@mail.ru"`
- `"user@mail.ri"`
- `"user@fmdd.ru"`
- `"user@mail"`
- `"^&@^8=*.&*"`

---

### 4. Регистрация с паролем короче 6 символов

**Предусловие:** Сервер запущен.

**Шаги:**
1. Отправить POST-запрос на `/auth/register` с телом `{"email": "user@mail.ru", "password": "<password>"}`

**Ожидаемый результат:**
- Код ответа: `422 Unprocessable Entity`
- В `detail[0].msg` содержится текст `"String should have at least 6 characters"`

**Наборы данных (password):**
- `""` (пустая строка)
- `"12345"` (5 символов)

---

### 5. Регистрация с паролем длиннее 30 символов

**Предусловие:** Сервер запущен.

**Шаги:**
1. Отправить POST-запрос на `/auth/register` с телом `{"email": "user@mail.ru", "password": "строка_из_31_символа"}`

**Ожидаемый результат:**
- Код ответа: `422 Unprocessable Entity`

---

### 6. Регистрация с пустым телом запроса

**Предусловие:** Сервер запущен.

**Шаги:**
1. Отправить POST-запрос на `/auth/register` с пустым JSON-объектом `{}`

**Ожидаемый результат:**
- Код ответа: `422 Unprocessable Entity`
- В ответе две ошибки: для поля `email` и для поля `password`
- В каждой ошибке `msg` содержит `"Field required"`, `loc` указывает на соответствующее поле

---

### 7. Регистрация с некорректным JSON

**Предусловие:** Сервер запущен.

**Шаги:**
1. Отправить POST-запрос на `/auth/register` с телом `'{"email": "user@mail.com", "password": }'` и заголовком `Content-Type: application/json`

**Ожидаемый результат:**
- Код ответа: `422 Unprocessable Entity`
- В `detail[0].msg` содержится текст `"JSON decode error"`

## Логин (POST /auth/login)

### 8. Успешный вход с корректными email и паролем

**Предусловие:** Сервер запущен. БД пуста.

**Шаги:**
1. Отправить POST-запрос на `/auth/register` с телом `{"email": "user@mail.ru", "password": "password"}` — зарегистрировать пользователя
2. Отправить POST-запрос на `/auth/login` с телом `{"email": "user@mail.ru", "password": "password"}`

**Ожидаемый результат:**
- Первый запрос: код `201 Created`, пользователь создан
- Второй запрос: код `200 OK`
- В теле ответа присутствует поле `token` с непустой строкой
- Токен валидный: при декодировании поле `user_id` совпадает с `id` пользователя из шага 1
